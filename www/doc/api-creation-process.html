Roundtrip command

A server command is implemented in a library, commonly where SQL is performed. Outside of this library, there is a common process to roundtrip the command such that it is made accessible. This section describes that external process.

Ingredients

  • Server Handlers and HTML pages that will provide access to the new command.
  • A form to submit the command.
  • Library to implement the command.
  • Module to contain the server handler.

Directions

Command Request/Response

A page that links to the command form is made accessible.

GET request

A form is GET requested that submits a request to invoke the server command.

GET request

Form submission is POST'd to a server handler, defined in a module.

POST request

The module invokes the server command, defined in a library.

-

The server command returns a status code.

The module writes a response for the client.

POST response

Make API endpoint available to clients

Ingredients

Directions

Add SQL field to existing table

Ingredients

  • Module that creates the table

Directions

  1. Update the create table definition with the new field definitions.
  2. Update the 'update module tables' definition to add the new field and bump the table version number.

Create a new SQL table

Ingredients

  • Table name
  • Field list

Directions

  1. Add a new create table definition with the given field definitions.

Add SQL statement to server

Ingredients

  • SQL statement.
  • Destination module.

Directions

  1. Add SQL statement to module SQL statements array. e.g. ~/src/parsegraph/<module>/<module>.c
  2. Add that same SQL statement to the server's prepared SQL statements array. e.g. ~/src/parsegraph/server/src/httpd.conf.in
