parsegraph Lexer

Based on 'Language Implementation Patterns' by Terence Parr.

new parsegraph_Lexer(input)

Creates a lexer for the given input string.

.consume()

Consumes the next token, returning true if a token was found, or false if there are no more tokens.

.c()

Returns the current token's character value.

.cc()

Returns the current tokens' character code.

.match(candidate)

Skips the current token if it is the given candidate.

Throws an error if the candidate matches the given string.

parsegraph_parse(str)

Parses the given string according to the built-in lexer definition, returning an array of parsed tokens.

parsegraph_parse(str, callback, thisArg)

Parses the given string according to the built-in lexer definition. Callback is invoked for each parsed token.
